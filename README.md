# End-to-End Banking Grievance Classification Pipeline

### 📌 Project Overview
This repository contains an automated data mining pipeline designed to ingest, clean, and systematically classify unstructured consumer complaint data dumps. The system isolates customer pain points and flags high-priority operational issues for banking institutions.

### 🛠️ Tech Stack
- **Data Ingestion & Cleaning:** Python (Pandas, NumPy)
- **Data Querying & Aggregation:** SQL (MySQL)
- **Trend Visualization:** Matplotlib & Seaborn
- **Version Control:** GitHub

### 🚀 Key Deliverables & Implementation
- **Automated Text Processing:** Built a Python pipeline to strip whitespace, handle missing values, and normalize raw string complaint text to lowercase for reliable parsing.
- **Keyword Extraction & Prioritisation:** Implemented custom logic to scan text and automatically flag high-priority triggers like "unauthorized digital transactions" and "ATM discrepancies".
- **Advanced SQL Querying:** Designed relational database schemas and utilized SQL Window Functions to segment and track dispute volumes across multiple bank branches.
